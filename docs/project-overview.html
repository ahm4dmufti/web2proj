Controllers

ProductController app/Http/Controllers/

Manages the antique gallery product catalog. Public landing page + full authenticated CRUD.

index() Public landing — top 3 by price (featured) + 5 most recent
all() Authenticated full inventory table, ordered by latest
create() Show the add-product form
store() Validate via ProductRequest & persist new product
edit() Show edit form pre-filled with product data
update() Validate & update product; handles unchecked checkbox for is_available
destroy() Delete product and redirect with flash success
TeamController app/Http/Controllers/Teams/

Handles team lifecycle — listing, creation, editing, renaming, switching, and deletion.

index() List all teams the authenticated user belongs to
store() Create new team via CreateTeam action, auto-generate slug
edit() Show team detail with members, pending invitations & permissions
update() Rename team using DB transaction + lockForUpdate() to prevent race conditions
switch() Set user's current_team_id to the selected team
destroy() Delete team (cascade invitations & memberships), auto-switch other affected users to their personal team
TeamInvitationController app/Http/Controllers/Teams/

Manages the full invitation lifecycle — sending, cancelling, and accepting invites.

store() Create invitation with 3-day expiry & send email notification
destroy() Cancel a pending invitation (owner/admin only via Gate)
accept() Create membership from invitation inside DB transaction, switch user to team, mark accepted_at
TeamMemberController app/Http/Controllers/Teams/

Handles role changes and removal of existing team members.

update() Change a member's role (Owner/Admin permission required via Gate)
destroy() Remove member; blocks owner removal; switches removed user back to personal team
ProfileController app/Http/Controllers/Settings/

User profile settings — view, update name/email, and delete account.

edit() Show profile form; passes mustVerifyEmail flag to frontend
update() Save name/email; clears email_verified_at if email changed
destroy() Log out, delete user, invalidate session & regenerate CSRF token
SecurityController app/Http/Controllers/Settings/

Password changes and Two-Factor Authentication management. Implements HasMiddleware for password confirmation.

middleware() Conditionally applies password.confirm middleware when 2FA confirmPassword option is on
edit() Show 2FA status — enabled, QR code state, recovery codes
update() Change password (rate-limited: 6 attempts/min via route)
